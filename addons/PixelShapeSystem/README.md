# PixelShapeSystem

`PixelShape` 是用于描述形状的对象，而不是用于存储瓦片、网格或地图数据的对象。

它的职责是：
- 描述一个形状覆盖了哪些像素点
- 以坐标、数组索引、列表、数组等形式输出这些像素点
- 供瓦片系统、网格系统等上层模块消费

这里的 `pixel` 是抽象像素，不限定具体载体。
在不同上下文中，它可以对应：
- 一个瓦片
- 一个网格单元
- 任何离散坐标上的最小元素

## 设计理念

- `PixelShape` 只负责描述形状，不直接承担缩放、镜像等可选能力
- 可选能力应通过额外接口或具体实现扩展，而不是强行塞进所有形状基类
- 默认实现优先简单、直接、高性能
- 需要几何变换能力时，再提供专门的变体类型

## 命名约定

### 变换能力约定
支持镜像、大小扩展等几何变换能力的形状增加Transformable前缀

### 点集形状例子

- `PointSetShape`
  - 默认的点集形状
  - 面向高性能使用
  - 为静态点集版本
  - 构造完成后不再允许追加点
  - 不支持镜像、大小扩展等几何变换能力

- `MutablePointSetShape`
  - 动态点集版本
  - 支持 `AddPoint` 与 `AddPoints`
  - 适合需要持续增量构建点集的场景

- `PointListShape`
  - 静态点列表版本
  - 保留输入顺序与重复点
  - 不做去重校验，适合脏且快的读取场景

- `MutablePointListShape`
  - 动态点列表版本
  - 支持 `AddPoint` 与 `AddPoints`
  - 仅以列表作为核心容器
  - 保留输入顺序与重复点，不做去重校验

- `TransformablePointSetShape`
  - 预留给支持镜像、大小扩展等几何变换能力的点集形状
  - 与默认静态版本、动态版本分开建模，避免把变换成本带入所有点集
